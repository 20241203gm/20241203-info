# PRD 문서: 20241203.info

## 1. 기획의도

### 20241203.info 프로젝트는 2024년 12월 3일 대한민국에서 윤석열에 의해 발생한 계란 사건을 기록하고 보존하기 위해 시작되었습니다.
### 1) 왜 계란인가요?
- 계엄이라고 하는데, 실상은 내란이었습니다. 계엄이라는 이름의 내란. 줄여서 계란입니다.
- 계란으로 바위치기라며, 우리 사회는 세상을 바꾸려는 이들을 말리거나 조롱했습니다. 하지만 이제 누가 계란입니까? 우리가 바위고, 저들이 계란입니다.
- (아 근데 계란은 죄가 없어요. 계란아 미안해)
### 2) 무엇을 하나요?
- 계란 관련 사건을 철저하게 기록합니다
- 계란을 시도한 윤석열과 그 일당은 물론이며, 이후 윤석열의 대통령 직위 박탈 과정에서 그들을 옹호하거나 직위 박탈을 지연시킨 이들을 기록합니다
### 3) 그래서 어떻게 할건가요?
- 기록의 힘을 믿습니다. 기록해야 기억할 수 있고, 기억이 변화의 첫걸음이라고 생각합니다.
- 2에 해당하는 인물들이 대한민국 정치에 발 붙이지 못하는 날까지 이 페이지를 운영합니다
### 4) 마지막으로
- 비록 체제에 줄곧 외면당하고 배제되어왔을 뿐이더라도, 그것이 나와 내 이웃의 삶의 근간이기에, 좌고우면하지 않고 거리로 뛰어나가 민주주의를 외친 시민들의 싸움과, 이들이 열어낸 미래를 기억하고자 합니다

## 2. 메뉴구성

1. **계란타임라인**
    - 계엄령과 이후 정치상황에 대한 시간순 사건 정리
    - 주요 사건별 상세 정보 제공
    - 관련 증거 자료 링크
        
2. **계란위키**
    - 타임라인과 각 인물들의 언행을 데이터베이스화
    - 문서, 뉴스 기사, 유튜브 클립 등 증거 자료 포함
    - 체계적인 정보 구조화 및 검색 기능

3. **광장의 목소리**
    - 스크롤 기반 스토리텔링으로 시민 활동 아카이브 제공
    - 배경 이미지/영상과 함께 시민들의 증언 강조
    - 주요 사건과 시민 활동 연계


## 3. 제작 방법

1. **데이터베이스 구축**
    - 구글시트를 이용하여 데이터 저장 및 관리.
    - 시트 내 각 열을 고유 ID, 제목, 내용, 날짜, 증거 자료 링크 등으로 구성.
    - 스크립트(API 또는 Apps Script)를 사용하여 구글시트 데이터를 JSON 형식으로 제공.
        
2. **위키형식 출력**
    - 노션 데이터베이스와 구글시트를 연동.
    - API 연동 또는 Zapier를 활용하여 구글시트에서 데이터를 자동으로 노션에 업데이트.
    - 노션의 퍼블릭 페이지 기능을 이용하여 위키 스타일로 출력.
        
3. **코딩 최소화**
    - HTML과 CSS를 활용한 간단한 프런트엔드 제작.
    - 구글 사이트(Google Sites) 또는 노션을 통해 웹페이지 구성.
        
4. **타임라인 서비스 활용**
    - Timeline.js 또는 Knight Lab Timeline을 이용하여 타임라인 구현.
    - JSON 데이터를 Timeline.js 형식에 맞춰 변환하여 시각화.
    - 타임라인에 필요한 이미지 및 동영상은 구글 드라이브 또는 유튜브 링크 활용.
        
5. **데이터 제출 및 관리**
    - 구글 폼을 통해 사용자 데이터 제출 기능 구현.
    - 제출된 데이터는 구글시트에 자동 기록.
    - 관리자가 데이터를 검토 후 승인 처리.
        
6. **웹페이지 제작 및 도메인 연결**
    - 구글 사이트 또는 노션을 사용하여 웹페이지 제작.
    - 도메인 구입 후 도메인 포워딩 설정.
    - HTTPS 지원 도메인 연결 서비스(예: Cloudflare) 활용.
        
7. **후원 시스템 구축**
    - 플랫폼(예: 텀블벅, 카카오페이, PayPal 등)을 활용한 후원 페이지 연결.
    - 후원 내역과 사용 계획을 투명하게 공개.

## 4. 서버 구성

1. **웹 호스팅**
    - Vercel 또는 Netlify
        - 무료 티어로 시작 가능
        - 자동 HTTPS 지원
        - GitHub 연동으로 자동 배포
        - 서버리스 함수 지원

2. **데이터베이스**
    - Supabase (PostgreSQL)
        - 무료 티어: 500MB
        - 실시간 데이터 지원
        - 인증 시스템 내장
        - REST/GraphQL API 자동 생성

3. **미디어 저장소**
    - Cloudinary
        - 무료 티어: 1GB 저장공간
        - 이미지/비디오 최적화
        - URL 기반 이미지 변환
    
4. **프론트엔드 스택**
    - Next.js
        - React 기반 프레임워크
        - SSR/SSG 지원
        - API 라우트 내장
    - GSAP + Intersection Observer
        - 스크롤 기반 애니메이션
        - 요소 등장 효과
    
5. **데이터 수집 및 관리**
    - Google Sheets API
        - 데이터 입력/관리용
        - 관리자 승인 프로세스
    - 간단한 서버리스 함수
        - Vercel Edge Functions
        - SNS 데이터 수집 및 가공

6. **모니터링**
    - Vercel Analytics
        - 기본 성능 모니터링
        - 사용자 행동 분석
    - Sentry 무료 티어
        - 에러 트래킹

예상 월 비용:
- Vercel/Netlify: 무료
- Supabase: 무료
- Cloudinary: 무료
- 도메인: 연 $10~20
- (선택) Sentry: 무료

총 예상 비용: 도메인 비용 외 거의 들지 않음

장점:
1. 서버 관리 불필요
2. 자동 스케일링
3. 높은 안정성
4. 개발 복잡도 감소
5. 인터랙티브 요소 유지

단점:
1. 일부 실시간 기능 제한
2. 무료 티어 사용량 제한
3. 커스터마이징 제약

필요시 나중에 AWS나 GCP로 마이그레이션 가능합니다.

### 개발 환경 설정

1. 저장소 클론
```bash
git clone https://github.com/your-username/20241203-info.git
cd 20241203-info
```

2. 의존성 설치
```bash
npm install
```

3. 개발 서버 실행
```bash
npm run dev
```
